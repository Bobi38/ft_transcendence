100 Continue
Cette réponse provisoire indique que le client doit poursuivre sa demande ou ignorer la réponse si la demande est déjà terminée.

101 Switching Protocols
Ce code est envoyé en réponse à un Upgradeen-tête de requête du client et indique le protocole vers lequel le serveur bascule.

102 Processing Déprécié
Ce code était utilisé dans le contexte WebDAV pour indiquer qu'une requête avait été reçue par le serveur, mais qu'aucun statut n'était disponible au moment de la réponse.

103 Early Hints
Ce code d'état est principalement destiné à être utilisé avec l' Linken-tête, permettant à l'agent utilisateur de commencer à précharger les ressources pendant que le serveur prépare une réponse ou se préconnecte à une origine à partir de laquelle la page aura besoin de ressources.

Réponses réussies
200 OK
La requête a abouti. Le résultat et la signification de ce succès dépendent de la méthode HTTP :

GETLa ressource a été récupérée et transmise dans le corps du message.
HEADLes en-têtes de représentation sont inclus dans la réponse sans aucun corps de message.
PUTou POST: La ressource décrivant le résultat de l'action est transmise dans le corps du message.
TRACELe corps du message contient la requête telle que reçue par le serveur.
201 Created
La requête a abouti et une nouvelle ressource a été créée. Il s'agit généralement de la réponse envoyée après POSTcertaines requêtes PUT.

202 Accepted
La requête a été reçue mais n'a pas encore été traitée. Son comportement est non engageant, car le protocole HTTP ne permet pas d'envoyer ultérieurement une réponse asynchrone indiquant le résultat de la requête. Ce type de réponse est destiné aux cas où un autre processus ou serveur traite la requête, ou au traitement par lots.

203 Non-Authoritative Information
Ce code de réponse signifie que les métadonnées renvoyées ne sont pas exactement identiques à celles disponibles sur le serveur d'origine, mais proviennent d'une copie locale ou d'un serveur tiers. Ce type de réponse est généralement utilisé pour les copies miroir ou les sauvegardes d'une autre ressource. En dehors de ce cas précis, cette 200 OKréponse est préférable à ce statut.

204 No Content
Aucun contenu n'est à envoyer pour cette requête, mais les en-têtes sont utiles. L'agent utilisateur peut mettre à jour ses en-têtes en cache pour cette ressource avec les nouveaux.

205 Reset Content
Indique à l'agent utilisateur de réinitialiser le document qui a envoyé cette requête.

206 Partial Content
Ce code de réponse est utilisé en réponse à une requête de plage lorsque le client a demandé une ou plusieurs parties d'une ressource.

207 Multi-Status( WebDAV )
Transmet des informations sur plusieurs ressources, pour les situations où plusieurs codes d'état pourraient être appropriés.

208 Already Reported( WebDAV )
Utilisé à l'intérieur d'un <dav:propstat>élément de réponse pour éviter d'énumérer à plusieurs reprises les membres internes de plusieurs liaisons à la même collection.

226 IM Used( Encodage delta HTTP )
Le serveur a satisfait une GETrequête pour la ressource, et la réponse est une représentation du résultat d'une ou plusieurs manipulations d'instance appliquées à l'instance actuelle.

Messages de redirection
300 Multiple Choices
Dans la négociation de contenu pilotée par un agent , la requête peut recevoir plusieurs réponses, et l'agent utilisateur ou l'utilisateur doit en choisir une. Comme il n'existe pas de méthode standardisée permettant aux clients de sélectionner automatiquement une réponse, cette approche est rarement utilisée.

301 Moved Permanently
L'URL de la ressource demandée a été modifiée définitivement. La nouvelle URL est indiquée dans la réponse.

302 Found
Ce code de réponse signifie que l'URI de la ressource demandée a été modifiée temporairement . D'autres modifications pourraient y être apportées ultérieurement ; le client devra donc utiliser la même URI pour ses prochaines requêtes.

303 See Other
Le serveur a envoyé cette réponse pour indiquer au client d'aller chercher la ressource demandée à une autre URI via une GETrequête.

304 Not Modified
Ceci sert à la mise en cache. Cela indique au client que la réponse n'a pas été modifiée, afin qu'il puisse continuer à utiliser la même version mise en cache de la réponse.

305 Use Proxy Déprécié
Définie dans une version précédente de la spécification HTTP pour indiquer qu'une réponse demandée devait être obtenue via un proxy, cette méthode est désormais obsolète en raison de problèmes de sécurité liés à la configuration en bande d'un proxy.

306 unused
Ce code de réponse n'est plus utilisé, mais il est réservé. Il était utilisé dans une version précédente de la spécification HTTP/1.1.

307 Temporary Redirect
Le serveur envoie cette réponse pour indiquer au client de récupérer la ressource demandée à une autre URI en utilisant la même méthode que celle employée lors de la requête précédente. Cette réponse a la même signification que le 302 Foundcode de réponse, à ceci près que l'agent utilisateur ne doit pas modifier la méthode HTTP utilisée : si une méthode HTTP POSTa été utilisée lors de la première requête, une méthode HTTP POSTdoit également être utilisée lors de la redirection.

308 Permanent Redirect
Cela signifie que la ressource est désormais accessible de manière permanente à une autre URI, spécifiée dans l' Locationen-tête de réponse. La sémantique est identique à celle du 301 Moved Permanentlycode de réponse HTTP, à ceci près que l'agent utilisateur ne doit pas modifier la méthode HTTP utilisée : si une POSTrequête HTTP a été utilisée lors de la première, une POSTrequête HTTP doit également l'être lors de la seconde.

Réponses d'erreur du client
400 Bad Request
Le serveur ne peut pas ou ne veut pas traiter la requête en raison d'une erreur perçue comme une erreur du client (par exemple, une syntaxe de requête mal formée, un cadrage de message de requête invalide ou un routage de requête trompeur).

401 Unauthorized
Bien que la norme HTTP spécifie « non autorisé », cette réponse signifie sémantiquement « non authentifié ». Autrement dit, le client doit s'authentifier pour obtenir la réponse demandée.

402 Payment Required
Ce code était initialement destiné aux systèmes de paiement numérique, mais il est rarement utilisé et il n'existe aucune convention standard.

403 Forbidden
Le client ne dispose pas des droits d'accès au contenu ; autrement dit, il n'est pas autorisé à y accéder, le serveur refuse donc de lui fournir la ressource demandée. Contrairement à [référence manquante] 401 Unauthorized, le serveur connaît l'identité du client.

404 Not Found
Le serveur ne trouve pas la ressource demandée. Dans le navigateur, cela signifie que l'URL n'est pas reconnue. Dans une API, cela peut également signifier que le point de terminaison est valide, mais que la ressource elle-même n'existe pas. Les serveurs peuvent aussi envoyer cette réponse pour 403 Forbiddenmasquer l'existence d'une ressource à un client non autorisé. Ce code de réponse est probablement le plus connu en raison de sa fréquence d'utilisation sur le web.

405 Method Not Allowed
Le serveur connaît la méthode de requêteDELETE , mais celle-ci n'est pas prise en charge par la ressource cible. Par exemple, une API peut ne pas autoriser cette méthode sur une ressource, voire ne pas TRACEl'autoriser du tout.

406 Not Acceptable
Cette réponse est envoyée lorsque le serveur web, après avoir effectué une négociation de contenu pilotée par le serveur , ne trouve aucun contenu conforme aux critères définis par l'agent utilisateur.

407 Proxy Authentication Required
C'est similaire, 401 Unauthorizedmais l'authentification doit être effectuée par un proxy.

408 Request Timeout
Certains serveurs envoient cette réponse lorsqu'une connexion est inactive, même sans requête préalable du client. Cela signifie que le serveur souhaite fermer cette connexion. Cette réponse est beaucoup plus fréquente depuis que certains navigateurs utilisent des mécanismes de pré-connexion HTTP pour accélérer la navigation. Certains serveurs peuvent fermer une connexion sans envoyer ce message.

409 Conflict
Cette réponse est envoyée lorsqu'une requête entre en conflit avec l'état actuel du serveur. Dans le cadre de la création de sites web à distance via WebDAV , 409les réponses sont des erreurs envoyées au client afin que l'utilisateur puisse résoudre le conflit et soumettre à nouveau la requête.

410 Gone
Cette réponse est envoyée lorsque le contenu demandé a été définitivement supprimé du serveur, sans qu'aucune adresse de redirection ne soit disponible. Les clients doivent vider leur cache et supprimer les liens vers la ressource. La spécification HTTP prévoit l'utilisation de ce code d'état pour les services promotionnels à durée limitée. Les API ne doivent pas se sentir obligées d'indiquer les ressources supprimées avec ce code d'état.

411 Length Required
Le serveur a rejeté la requête car le Content-Lengthchamp d'en-tête n'est pas défini et le serveur l'exige.

412 Precondition Failed
Dans les requêtes conditionnelles , le client indique dans ses en-têtes des préconditions que le serveur ne remplit pas.

413 Content Too Large
Le corps de la requête dépasse les limites définies par le serveur. Le serveur peut fermer la connexion ou renvoyer un Retry-Afterchamp d'en-tête.

414 URI Too Long
L'URI demandée par le client est plus longue que ce que le serveur est disposé à interpréter.

415 Unsupported Media Type
Le format multimédia des données demandées n'est pas pris en charge par le serveur, qui rejette donc la requête.

416 Range Not Satisfiable
Les plages spécifiées dans l' Rangeen-tête de la requête ne peuvent être respectées. Il est possible que la plage soit supérieure à la taille des données de la ressource cible.

417 Expectation Failed
ExpectCe code de réponse signifie que le serveur ne peut pas satisfaire l'attente indiquée dans l' en-tête de la requête.

418 I'm a teapot
Le serveur refuse qu'on lui prépare du café avec une théière.

421 Misdirected Request
La requête a été adressée à un serveur incapable de répondre. Cela peut être dû à un serveur non configuré pour répondre à la combinaison de schéma et d'autorité figurant dans l'URI de la requête.

422 Unprocessable Content( WebDAV )
La requête était bien formulée mais n'a pas pu être suivie en raison d'erreurs sémantiques.

423 Locked( WebDAV )
La ressource à laquelle vous tentez d'accéder est verrouillée.

424 Failed Dependency( WebDAV )
La requête a échoué en raison de l'échec d'une requête précédente.

425 Too Early Expérimental
Indique que le serveur ne souhaite pas prendre le risque de traiter une requête susceptible d'être rejouée.

426 Upgrade Required
Le serveur refuse d'exécuter la requête avec le protocole actuel, mais pourrait l'accepter une fois que le client aura migré vers un autre protocole. UpgradeDans la réponse 426, le serveur envoie un en-tête indiquant le ou les protocoles requis.

428 Precondition Required
Le serveur d'origine exige que la requête soit conditionnelle . Cette réponse vise à prévenir le problème de « mise à jour perdue », où un client GETenregistre l'état d'une ressource, le modifie et PUTle renvoie au serveur, alors qu'entre-temps, un tiers a modifié cet état sur le serveur, ce qui entraîne un conflit.

429 Too Many Requests
L'utilisateur a envoyé trop de requêtes dans un laps de temps donné ( limitation du débit ).

431 Request Header Fields Too Large
Le serveur refuse de traiter la requête car ses en-têtes sont trop volumineux. Vous pouvez la soumettre à nouveau après avoir réduit la taille de ses en-têtes.

451 Unavailable For Legal Reasons
L'agent utilisateur a demandé une ressource qui ne peut être légalement fournie, telle qu'une page Web censurée par un gouvernement.

Réponses d'erreur du serveur
500 Internal Server Error
Le serveur rencontre une situation qu'il ne sait pas gérer. Cette erreur générique indique que le serveur ne trouve pas de 5XXcode d'état plus approprié pour répondre.

501 Not Implemented
Cette méthode de requête n'est pas prise en charge par le serveur et ne peut être traitée. Les seules méthodes que les serveurs sont tenus de prendre en charge (et qui ne doivent donc pas renvoyer ce code) sont `get` GETet `get` HEAD.

502 Bad Gateway
Cette réponse d'erreur signifie que le serveur, alors qu'il fonctionnait comme une passerelle pour obtenir la réponse nécessaire au traitement de la requête, a reçu une réponse invalide.

503 Service Unavailable
Le serveur n'est pas prêt à traiter la requête. Cela peut être dû à une maintenance ou à une surcharge du serveur. Veuillez noter qu'une page explicative du problème doit accompagner cette réponse. Cette réponse est destinée aux situations temporaires et l' Retry-Afteren-tête HTTP doit, si possible, indiquer le délai estimé de rétablissement du service. Le webmaster doit également veiller à la gestion des en-têtes relatifs à la mise en cache, car ces réponses temporaires ne doivent généralement pas être mises en cache.

504 Gateway Timeout
Ce message d'erreur est renvoyé lorsque le serveur fait office de passerelle et ne peut pas obtenir de réponse à temps.

505 HTTP Version Not Supported
La version HTTP utilisée dans la requête n'est pas prise en charge par le serveur.

506 Variant Also Negotiates
Le serveur présente une erreur de configuration interne : lors de la négociation de contenu, la variante choisie est configurée pour s’engager elle-même dans la négociation de contenu, ce qui entraîne des références circulaires lors de la création des réponses.

507 Insufficient Storage( WebDAV )
La méthode n'a pas pu être exécutée sur la ressource car le serveur est incapable de stocker la représentation nécessaire pour mener à bien la requête.

508 Loop Detected( WebDAV )
Le serveur a détecté une boucle infinie lors du traitement de la requête.

510 Not Extended
La requête du client déclare une extension HTTP ( RFC 2774 ) qui devrait être utilisée pour traiter la requête, mais l'extension n'est pas prise en charge.

511 Network Authentication Required
Indique que le client doit s'authentifier pour accéder au réseau.
